# 採用 super-engine + 腦庫組合，啟動 gbrain-companion 專案

> Mirror of `wiki/decisions/2026-05-16-gbrain-companion` in gbrain.
> Status: approved | Created: 2026-05-16

## 決策

建立 `gbrain-companion` 專案，以 super-engine 為 driver、gbrain 為知識層，組成自有 AI pipeline。

```
腦庫(gbrain) ↔ super-engine ↔ Gemini Web ↔ Ryan
```

核心定位：把已付費的 Gemini Pro 訂閱程式化利用，讓腦庫能透過 super-engine 驅動 Gemini Web 完成讀取、思考、寫回的閉環。

## 背景痛點

- Gemini 網頁版無法呼叫外部 API（沙盒限制）
- AI Studio 有 rate-limit bug（免費 tier 5 RPM，multi-step 不穩定）
- Gemini CLI 體驗差，不適合自動化
- 現有 Gemini Pro 訂閱未被程式化利用

## 替代方案評估

| 方案 | 結論 |
|------|------|
| 繼續用 Claude.ai crawler | 備用，但綁定單一平台 |
| 用 Gemini API | 否決，需額外付費 |
| 用 project-golem | 否決，架構太重 |
| 自建 gbrain-companion | **選定**，最小依賴、完全自主 |

## 預期價值

- 跨 AI 沉澱自動化
- 腦庫自我成長閉環
- 戰略獨立性，不依賴任何 AI 平台政策
- 成本最佳化，現有 Pro 訂閱完全利用

## 不做的事

不開源、不做 multi-provider、不做 agent 框架、不做產品化、不追求 100% 自動化。

## MVP 路線

Phase 1–4，預計 4 個週末完成核心功能。Phase 5（Telegram bridge）選配。

## 退場條件

6 個月後若 Gemini Web DOM 維護成本 >16h/月、大規模封號、或出現更好替代方案，降級為純參考實作。
